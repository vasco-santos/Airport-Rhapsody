Detailed Description

Problem name: Airport rhapsody.

(interface file)

Concept: António Rui Borges

Synchronization based on semaphores and shared memory. Implementation with SVIPC.

Definition of the operations carried out by the bus driver thread:

  • has day's work ended
  • announcing bus boarding
  • go to departure terminal
  • park the bus and let passengers off
  • go to arrival terminal
  • park the bus.
Author
António Rui Borges - December 2013

Function Documentation

void announcingBusBoarding ( void  )

Announcing bus boarding.

The bus driver checks if the queue is empty. If it is, he issues an error message. He then proceeds to summon passengers in the queue to board the bus. If some passenger identity is unknown, he issues an error message. He finally waits for the boarding to be complete.

No state should be saved.

void goToArrivalTerminal ( void  )

Go to arrival terminal.

The bus driver takes the bus back to the arrival transfer terminal.

State should be saved.

State should be saved.

bool hasDaysWorkEnded ( void  )

Has days work ended.

The bus driver keeps waiting for passengers to transfer until his day's work has come to an end. He only proceeds if his day's work is indeed finished or there are passengers needing to be serviced.

No state should be saved.

Returns
true, if the day has come to the end
false, if there are passengers needing to be serviced
void parkTheBus ( void  )

Park the bus.

The parks the bus at the arrival transfer terminal.

State should be saved.

void parkTheBusAndLetPassOff ( void  )

Park the bus and let passengers off.

The bus driver checks if the bus is empty or overcrowded. If it is, he issues an error message. He then proceeds to summon passengers in the bus to exit. If some passenger identity is unknown, he issues an error message. He finally waits for the exit to be complete.

State should be saved.
